# Проект AgentStatuser: Система мониторинга Gemini CLI на ESP32

## Архитектура решения
Система построена на событийно-ориентированной модели (Producer-Consumer) для исключения блокировок основного процесса Gemini:
1. **Hooks (Producer):** Gemini CLI вызывает `scripts/hook.py` при наступлении событий.
2. **Broker (Redis):** Docker-контейнер `gemini-redis` (порт 6379) принимает JSON-события через очередь `gemini_events`.
3. **Bridge (Consumer):** Фоновый процесс `scripts/bridge.py` читает Redis, определяет проектную папку по PPID и пересылает данные в Serial. Конфигурация (Redis, Serial) берется из `scripts/.env`.
4. **Display (ESP32):** Плата LilyGo TTGO T-Display (ESP32-D0WDQ6) отображает до 4-х динамических виджетов.

## Управление и ориентация (Buttons)
На плате задействованы две физические кнопки для управления ориентацией экрана:
- **Верхняя кнопка (GPIO 35):** Устанавливает стандартную альбомную ориентацию (`tft.setRotation(1)`).
- **Нижняя кнопка (GPIO 0):** Разворачивает экран на 180 градусов (`tft.setRotation(3)`).
При смене ориентации интерфейс перерисовывается автоматически.

## Логика виджетов (LRU-стиль)
На экране отображаются 4 последних активных источника:
- Если данных нет, отображается статус **"Waiting"**.
- Если приходит сообщение от существующего источника, он обновляется и перемещается в **самый верхний слот**.
- Если приходит сообщение от нового источника, он добавляется в топ, сдвигая остальные вниз. Если все 4 слота заняты, самый старый (нижний) удаляется.
- Код `K` (Kill) удаляет виджет источника с экрана.

## Протокол и Связь
- **Каналы:** Поддерживается передача через USB Serial и **Bluetooth Serial (SPP)**.
- **Имя Bluetooth:** `AgentStatuser`.
- **Протокол:** `@L:C:Folder:Status#` (сообщения) и `@Q:L:F:P#` (квоты).
- **C (Color Code):**
  - `B` (Blue): Ready / SessionStart.
  - `Y` (Yellow): BeforeModel / BeforeTool.
  - `G` (Green): AfterModel.
  - `R` (Red): ToolPermission / Errors.
  - `K` (Kill): Удаление виджета.

## Файловая структура
- `ESP32-D0WDQ6/main.cpp`: Прошивка (Arduino/C++). Имя папки соответствует чипу.
- `scripts/hook.py`: Оптимизированный хук (Raw Socket RESP).
- `scripts/bridge.py`: Python-мост с дедупликацией, маппингом PPID, PID-защитой и чистым завершением.
- `platformio.ini`: Настройки сборки. Параметр `src_dir` указывает на `ESP32-D0WDQ6`.

## Правила работы
1. При изменении прошивки сохранять совместимость с протоколом.
2. Перед обновлением `bridge.py` убивать старые процессы: `pkill -f bridge.py`.
3. Скрипт `bridge.py` использует `bridge.pid` для предотвращения запуска дубликатов.
4. Для работы через Bluetooth укажите соответствующий порт в `scripts/.env`.
